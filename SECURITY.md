# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in autopus-adk, please report it responsibly.

**Email**: security@autopus.co

**Response SLA**:
- Acknowledgment: within 48 hours
- Initial assessment: within 5 business days
- Patch release: within 7 days for critical vulnerabilities

**Do NOT**:
- Open a public GitHub issue for security vulnerabilities
- Disclose the vulnerability publicly before a patch is available

## Scope

The following are in scope:
- autopus-adk CLI binary
- Worker authentication and credential storage
- SecurityPolicy validation and enforcement
- Secret scanning and redaction

The following are out of scope:
- The Autopus platform server (report to platform security team)
- Third-party dependencies (report to upstream maintainers)

## Recognition

We appreciate responsible disclosure and will acknowledge security researchers in our release notes (with permission).
